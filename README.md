# K8 退货处理系统 V7（手机拍照修复版）

V7修复了手机点击“手机相机拍照”后，照片没有出现在“本件照片预览”的问题。

## 本次修复

### 1. 避免两个摄像头同时占用

打开手机原生相机前，系统会先停止网页实时摄像头。

拍照返回网页后，再恢复实时摄像头。这样可以避免部分 iPhone 和 Android 设备出现：

- 拍照后网页没有收到文件
- 相机黑屏
- 照片没有保存
- 浏览器页面被系统回收

### 2. 增加手机照片格式兼容

系统现在会：

1. 优先使用 `createImageBitmap`
2. 失败后自动改用普通图片解码
3. 对 HEIC/HEIF 给出明确提示
4. 对 JPEG、PNG 和手机相机照片转换成适合系统保存的 JPEG

### 3. 降低手机内存压力

手机照片可能达到 12MP、24MP 或更高。

V7在手机端将最大边控制为 2048 像素，避免 iPhone Safari 因画布内存不足而无反应。该清晰度仍足够查看：

- 面单文字
- SKU
- 商品整体
- 外观瑕疵

### 4. 保存过程增加错误提示

原版本在照片转换失败时，界面可能没有任何提示。

V7会明确显示：

- 正在处理照片
- 照片已保存
- 本件已经保存多少张
- HEIC不兼容
- 图片过大
- 浏览器未能写入照片数据库

### 5. 摄像头下方显示即时缩略图

手机拍照返回后，摄像头下方会立即出现：

- “已保存 X 张”
- 最近3张照片的缩略图
- 点击缩略图可以放大查看

原有“本件照片预览”区域也会同步更新。

## iPhone HEIC设置

如果系统提示无法转换 HEIC：

1. 打开 iPhone“设置”
2. 进入“相机”
3. 进入“格式”
4. 选择“兼容性最佳”

之后新拍的照片会使用 JPEG，更适合网页和 Excel。

## 更新已部署网站

如果之前已经上传到 Netlify：

1. 解压 V7
2. 登录 Netlify
3. 打开原来的项目
4. 进入 Deploys
5. 重新拖入整个 V7 文件夹
6. 等待发布完成
7. 手机重新打开网页
8. 如果仍显示旧版，关闭桌面图标并重新打开，或清除 Safari/Chrome 网站缓存

## 重要说明

手机和电脑的数据仍然保存在各自浏览器中，尚未进行云端同步。
